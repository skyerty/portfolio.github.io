Projet GSB

Conception et déploiement d'une infrastructure 3-tiers segmentée (Internet / DMZ / LAN / Base de données) pour garantir la disponibilité et la confidentialité des données frais des visiteurs médicaux.

DMZ + VLAN Firewalls & ACL Authentification AD PRA & sauvegardes

Visiteurs médicaux actifs

+450

Gestion des connexions simultanées via HTTPS vers une application web hébergée en local

Objectifs

01 • Centralisation

Unifier la collecte des comptes-rendus et notes de frais afin de disposer d’une source unique et exploitable par les équipes siège.

02 • Sécurité

Assurer un accès distant chiffré (HTTPS, bastion) tout en segmentant les données sensibles dans des zones privées.

03 • Continuité

Automatisation, sauvegardes et supervision afin de maintenir le service pour les visiteurs même en cas d’incident.

Outils techniques

Infrastructure

Cisco, pfSense, HAProxy, OpenVPN

Systèmes

Windows Server 2022, Debian 12

Accès & support

Apache Guacamole, GLPI, URBackup

Observabilité

Zabbix, MariaDB

Cisco pfSense HAProxy OpenVPN Windows Server Debian Guacamole GLPI Zabbix URBackup MariaDB
Architecture cible

Segmentation multi-zones et flux maîtrisés

Retour aux projets

Internet & DMZ

  • • Pare Feu pfsense avec 6 interface (WAN, LAN Serveurs, LAN PC, DMZ, Zone Interne BDD, Zone d'administration)
  • • Reverse proxy, pour protéger le serveur web et socker la page web
  • • Supervision temps réel via Zabbix
  • • Bastion d'administration isolé dans une autre interface

Explication de l'infrastructure

  • • LAN Serveurs: AD DS, DNS, DHCP, GLPI, URBackup
  • • DMZ : Serveur Web protéger par le HAProxy
  • • Zone Interne BDD : MariaDB répliqué
  • • Zone d'administration : Bastion
  • • LAN PC : Tous les hôtes de Paris
