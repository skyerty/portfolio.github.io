Transformation numérique de Grégoire Construction

Mise en place d'une infrastructure IT complète, sécurisée et centralisée pour l'entreprise Grégoire Construction.

Infrastructure Active Directory

L'Active Directory (AD) a été mis en place pour centraliser les utilisateurs et sécuriser l’accès aux ressources du réseau. Le domaine actif est gregoireconstruction.fr.

  • Création structurée des utilisateurs et groupes par services.
  • Comptes personnalisés avec des privilèges spécifiques.
  • Organisation hiérarchique des données avec des répertoires personnels et de groupes structurés.

Services Réseau Fondamentaux & GPO

Les services essentiels DNS et DHCP ont été configurés pour assurer la connectivité, complétés par une stratégie de sécurité et d'administration centralisée via les GPO (Group Policy Objects).

Services Réseau :

  • Service DNS : DNS 1 : 192.168.37.201, DNS 2 : 192.168.10.1
  • Service DHCP : Plage d'attribution de 192.168.37.205 à 192.168.37.250

Configuration des GPO :

  • Sécurité renforcée : Blocage des ports USB (selon services) et restrictions va le Panneau de configuration
  • Mappage automatique des lecteurs réseau personnels et de services
  • Déploiement logiciel automatisé (Chrome et agent GLPI)
  • Personnalisation : Fond d'écran uniforme pour l'identité visuelle

Serveur Web (Debian 12) & Solution GLPI

Un serveur Debian 12 a été configuré pour héberger l'image publique de l'entreprise et centraliser la gestion du parc informatique

Serveur Web :

  • Installation d'Apache 2 avec configuration de virtuals hosts
  • Configuration du nom de domaine www.gregoireconstruction.fr avec certificats de sécurité (via hôte DNS)
  • Déploiement d'un site vitrine

Solution GLPI :

  • Gestion centralisée des incidents et inventaire automatisé des équipements (via Agent GLPI)
  • Authentification simplifiée via les identifiants Active Directory
  • Tableau de bord pour la visualisation de l'état du parc

Base de Données & Stratégie de Sauvegarde

La pérennité et la sécurité des données critiques sont assurées par un serveur de Base de Données isolé et une stratégie de sauvegarde robuste

Serveur de Base de Données (Debian 12) :

  • Moteur MariaDB, performant et sécurisé
  • Centralisation de toutes les données applicatives pour un point de gestion unique
  • Isolement pour renforcer la sécurité contre les menaces (ex: ransomwares)

Stratégie de Sauvegarde :

  • Scripts Bash automatisés pour la sauvegarde des serveurs web et base de données
  • Transfert sécurisé vers un NAS (OpenMediaVault) via SCP avec clés RSA
  • Le script automatise le dump de la base de données GLPI, la récupération des fichiers via SCP, la compression en `.tar.gz` et le transfert vers le NAS distant

Conclusion

Cette nouvelle infrastructure IT offre à Grégoire Construction une solution complète et sécurisée, garantissant une sécurité renforcée contre les ransomwares, une efficacité opérationnelle grâce à GLPI et l'AD, ainsi que la pérennité des données grâce à une stratégie de sauvegarde robuste
